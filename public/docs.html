Welcome to Ketebe

Ketebe Engine is a multi-platform game development studio built with Electron. It lets you build 2D and 3D games and export them to Windows desktop, Android, and web — all without leaving the studio.

Philosophy: "Visual where possible, code where necessary." Every tool writes plain JSON or ES module JS. No proprietary binary formats.

Engine Types

Engine IDStyleRendererUse Case
rpg-topdown2D top-downCanvas 2DRPG, adventure, Zelda-like
platformer-2d2D side-scrollCanvas 2DPlatformer, action side-scroller
iso-pixel2.5D isometricCanvas 2DIsometric strategy, city builder
topdown-3d3D top-downThree.js WebGLRTS, strategy, ARPG
fps-3d3D first-personThree.js WebGLFPS, dungeon crawler
platformer-3d3D third-personThree.js WebGL3D platformer, adventure

Tech Stack

  • Electron — desktop wrapper and studio shell
  • Express.js — embedded web server for file I/O and API
  • Three.js r158+ — WebGL renderer for 3D engines
  • cannon-es — rigid-body physics for 3D engines (pure JS, no WASM)
  • Capacitor — Android packaging

Visual Style

All 3D engines use a low-poly + voxel aesthetic: flat-shaded geometry, palette-driven colors, no photo-realistic textures. Optional Minecraft-style texture atlases (TextureAtlas3D) are supported as an opt-in feature.

Quick Start

1. Install & Launch

npm install
npm start          # launches Electron studio

2. Create a Project

  1. Click NEW PROJECT on the dashboard.
  2. Enter a project name and pick an engine type.
  3. A folder is created in projects/YourProjectName/.

3. Open the Right Editor

From the dashboard, select your project and click the editor button. The studio opens the editor matching the engineType in ketebe.json.

4. Design & Playtest

Build your level, then click ▶ PLAYTEST to launch in a preview window. Logs route back to the editor console automatically.

5. Build for Distribution

# All platforms
npm run build:game "ProjectName"

# Electron only
npm run build:game "ProjectName" electron

# Android only
npm run build:game "ProjectName" android
Output is written to dist/game/. For Electron builds, a packaged .exe is in dist/game/release/.

IsoPixel Studio

A specialized isometric 2.5D editor for creating detailed pixel art worlds with depth. Engine type: iso-pixel.

Core Features

  • Isometric Grid: True isometric perspective — tiles are 2:1 diamonds.
  • Elevation Stacking: Place blocks at different Z heights to build 3D-looking structures.
  • Depth Sorting: Per-tile depth sort using z*(W+H+2)+(x+y) formula — no z-fighting on elevated tiles.
  • Foliage Generator: Procedural grass, trees, and bushes with configurable density.
  • NPC & Path Placement: Isometric character sprites with path-following support.
  • Day/Night Preview: Dynamic lighting preview using palette shift.

AI Behavior Editor

Design AI "brains" using a behavior tree / state machine editor. Saved to data/brains/*.json.

Node Types

States (3D engines)

3D enemies use an FSM with these states:

  • Patrol → cycles through waypoints
  • Alert → heard a sound, searching
  • Chase → player in sight, closing distance
  • Attack → within attack range, firing/striking
  • Cover → seeking a cover point while reloading

Script Editor

Full-featured Monaco-based code editor for writing game logic in ES6 JavaScript. Scripts live in data/logic/ and are served by /api/logic/js/:name.

Script API

export async function runLogic(entity, game, ui) {
    game.log('Script running...');

    // Walk entity to a position
    await entity.walkTo(100, 200);

    // Play a sound
    game.audio.play('explosion');

    // Show dialogue
    game.dialogueSystem.start('boss_intro');

    // Read/write flags
    game.setFlag('boss_defeated', true);
    if (game.getFlag('key_found')) {
        game.loadLevel('castle_interior');
    }
}

Rendering Pipeline

WebGLRenderer (Three.js r158+)
  └─ EffectComposer
      ├─ RenderPass           (main scene)
      ├─ OutlinePass          (selected/highlighted objects)
      └─ ShaderPass           (CelQuantize — posterize to palette colors)

Three.js Import Pattern

All shared modules use this dual-import pattern to avoid "multiple instances" warnings when editors load three.min.js as UMD while engines use three.module.js as ES module:

import * as _THREE_MOD from '/lib/three/three.module.js';
const THREE = (typeof globalThis !== 'undefined' && globalThis.THREE) || _THREE_MOD;

MeshPhongMaterial Convention

All 3D engines use MeshPhongMaterial({ shininess: 0, flatShading: true }) — NOT MeshLambertMaterial. Lambert does not support flatShading on Three.js r128+.

Collectible Persistence

Collected IDs are stored in the save via Save3D.js. On level reload, CollectibleSystem3D.markCollected(id) disables the pickup trigger per saved ID. Items still render (cosmetic quirk by design) but cannot be re-collected.

Tileset Atlas System

Optional Minecraft-style UV texturing for all 3D engines. Module: public/engines/shared/TextureAtlas3D.js. Purely opt-in — solid palette colors remain the default.

Enable / Disable

  • In any 3D editor: click the 🧱 TILESET button in the toolbar.
  • In ketebe.json: set "tilesetEnabled": true.
  • In code: call engine.enableTileset() / engine.disableTileset().

TextureAtlas3D API

import TextureAtlas3D from '/engines/shared/TextureAtlas3D.js';

const atlas = new TextureAtlas3D();     // uses procedural default
await atlas.loadAsync(THREE);

const mat = atlas.getMaterial(THREE);   // one shared MeshStandardMaterial
atlas.applyBlockUVs(boxGeo, 'GRASS');   // write UV for all 6 faces of a BoxGeometry
MethodDescription
loadAsync(THREE)Load atlas texture + config. Must await before use.
getMaterial(THREE)Returns the cached MeshStandardMaterial for the atlas
applyBlockUVs(geo, blockType)Writes UV attribute on any 6-face BufferGeometry
getUVRect(tileIndex)Returns {u0,v0,u1,v1} for tile at given atlas index
generateDefaultCanvas()Static. Draws 256×256 procedural pixel-art atlas in memory.

Tri-mesh / Low-poly Terrain

All three 3D editors support a TRI-MESH layer mode alongside the standard VOXEL mode. Both layers can co-exist in the same level.

Mode Toggle

In any 3D editor, open the GEO panel and click VOXEL or TRI-MESH. Switching to TRI-MESH generates a flat low-poly mesh that can be sculpted.

Sculpt Tools

ToolLMB Drag Effect
RaisePush vertices upward within brush radius
LowerPull vertices downward
FlatFlatten vertices toward Y=0

Brush radius and strength are adjustable via sliders. All sculpt tools use a squared falloff (smooth brush feel).

Dual-Layer Export

When saved, levels bundle BOTH layers: raw voxel grid + tri-mesh vertex/color data. At build time, voxel data is baked to GLTF via greedy meshing; tri-mesh geometry is included as-is. Both are merged into a single optimized scene GLTF.

const elevGrid = new Float32Array(20 * 20); // W*H elevation floats

Audio System

Web Audio API wrapper via AudioSpatial3D.js (3D engines) or the 2D engine audio layer.

Features

  • Reverb Zones: AudioListener + ConvolverNode per zone (caves = long reverb, open = dry)

EventBus — Cross-Tool Communication

File: public/shared/EventBus.js — Auto-instantiated as window.KetebeEventBus

The EventBus is the backbone of all inter-tool communication in Ketebe. It operates on three channels simultaneously: local listeners within the same window, cross-window postMessage for iframes and popups, and WebSocket broadcast for real-time sync between multiple editor tabs.

Core API

MethodSignatureDescription
on()(event, callback, opts)Subscribe to an event. Returns a listener ID string. opts.priority (higher = first), opts.once (auto-unsubscribes).
once()(event, callback)Subscribe for exactly one call, then auto-remove.
off()(event, idOrFn)Unsubscribe by listener ID string, or by original callback function reference.
emit()(event, data)Fire an event locally and broadcast it over WS + postMessage to all other windows.
getHistory(filter)(string|fn|null)Return up to last 100 events. Filter by event name glob or predicate function.
clearHistory()()Clear the event history ring buffer.
getStatus()()Returns { isConnected, reconnectAttempts, listeners[], historySize }

Usage Examples

const bus = window.KetebeEventBus;

// Subscribe to an event
const listenerId = bus.on('enemy:updated', (event) => {
    console.log('Enemy changed:', event.data.enemyId);
});

// Unsubscribe later
bus.off('enemy:updated', listenerId);

// Subscribe once (auto-removes after first call)
bus.once('project:loaded', (event) => {
    console.log('Project loaded:', event.data.name);
});

// Emit an event (goes to all tools)
bus.emit('skill:updated', { skillId: 'fireball', action: 'updated' });

// Wildcard listener — catches any event starting with 'asset:'
bus.on('asset:*', (event) => {
    console.log('Asset event:', event.type, event.data);
});

Event Object Structure

{
  "type":      "enemy:updated",            // event name
  "data":      { "enemyId": "goblin_01" }, // payload (any shape)
  "timestamp": 1741546402000,              // Unix ms when emitted
  "source":    "/enemy_editor_abc12"       // originating window path+id
}

Wildcard Patterns

Event names support * as a glob wildcard. * alone matches everything; asset:* matches asset:sprite:updated, asset:background:saved, etc.

Transport: WebSocket

On load, EventBus connects to ws://localhost:3000/ws. Any event you emit is serialized to JSON and sent over the socket to all other connected windows. Events arriving from remote sources are deserialized and dispatched to local listeners — but your own events are ignored (checked by source ID) to prevent echo loops. On disconnect, it auto-reconnects with exponential backoff (max 5 attempts, up to 30 s delay).

Transport: postMessage

Every emitted event is also broadcast via postMessage to three targets: the parent window (if inside an iframe), all child iframes, and window.opener (if opened as a popup). The message is wrapped as { sourceType: 'ketebe-event', event: {...} }. This is how the game window logs route back to the editor console.

Writing a Tool Integration

Follow this boilerplate pattern when integrating a new tool with the EventBus:

// At top of your tool's JS file
let eventBus, projectState, assetManager;

function initializeIntegration() {
    eventBus     = window.KetebeEventBus;
    projectState = window.KetebeProjectState;
    assetManager = window.KetebeAssetManager;

    if (!eventBus) return;

    // Listen for events from other tools
    eventBus.on('some:event', (event) => {
        console.log('[MyTool] Received:', event.data);
        handleUpdate(event.data);
    });

    console.log('[MyTool] EventBus connected');
}

// Call during DOMContentLoaded
document.addEventListener('DOMContentLoaded', () => {
    initializeIntegration();
});

// Broadcast changes to other tools
function broadcastMyUpdate(item, action = 'updated') {
    if (!eventBus) return;
    eventBus.emit('mytool:updated', {
        itemId: item.id,
        action,
        source: 'my_editor'
    });
}

Uniform Auto-Parsing

When the shader compiles, parseUniforms(src) scans the source for uniform declarations and renders matching controls:

uniform float u_intensity;  // → slider 0.0 – 1.0
uniform float u_pixelSize;  // → slider 1 – 32
uniform vec3  u_tint;       // → RGB color picker

Shader Entry Points

All shaders must define a main() entry using gl_FragCoord, u_resolution, u_time, and u_texture. The runtime injects these uniforms automatically.

EventBus Integration

Listens on shader:requested. Broadcasts shader:updated on save so 3D editors can hot-reload custom material shaders.
