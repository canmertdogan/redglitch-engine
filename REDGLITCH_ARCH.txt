  DETAILED COMPONENT BREAKDOWN & RECENT EVOLUTIONS (v9.2):

  1. THE EXPRESS SERVER MODULARIZATION:
     - `server.js` has been heavily refactored from a monolithic file into a clean `server/` directory containing modular `routes/`, `services/`, `middleware/`, and `websocket/` managers.
     - The IDE UI securely requests data and assets through `AssetManager.js`, which sanitizes relative paths client-side to prevent 404s, while the server intelligently routes between `projects/` and the core `public/` engine fallbacks.

  2. THE KETEBE AI MICRO SYSTEM (PHASE 2 RAG):
     - The architecture operates a Dual-Layer AI system. `server.js` acts as a proxy, piping heavy LLM inference (`/api/ai`) to a background Python backend (`backend/main.py`) running DeepSeek-Coder-1.3B.
     - Concurrently, the frontend (`public/ai/`) operates a fully local RAG engine. Using `embedding-worker.js` (Transformers.js v3), it generates embeddings dynamically in Web Workers to prevent UI locking.
     - Vectors are stored locally in an IndexedDB-backed Orama database (`vector-store.js`), seamlessly querying the generated `corpus.json`.

  3. SHARED KERNEL BACKBONE (`public/shared/`):
     - `EventBus.js` remains the heart of the engine, syncing internal state via WebSockets (Port 3000) across all decoupled IDE tabs, AI systems, and Game Loops.
     - `SharedProjectState.js` acts as the single source of truth for the active client, while `AssetManager` dictates resource fetching.

  4. THE ISOPIXEL v8 UPGRADES:
     - The old `worlds-grid` system was sunset, replaced by a universal, responsive `.asset-grid` & `.asset-card` layout across Prefabs, Worlds, and the newly added NPCs tab.
     - The Engine rendering loop removed occlusion culling entirely and disabled image smoothing, opting to brute-force tiles and sprites for maximized visual stability and crisp pixel art.
     - A dynamic Day/Night lighting cycle runs continuously in tandem with NPC spawning logic.
